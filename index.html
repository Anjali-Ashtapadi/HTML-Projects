CSS Position Properties

  • position - static
  • position - relative
  • position - fixed
  • position - absolute
  • position -sticky

Static

All HTML elements are positioned static by default. Static positioned elements are not affected by the top, bottom, left, and right properties.An element with position: static; is always positioned according to the normal flow of the page:


Relative

An element with position: relative; is positioned relative to its normal position in the document flow. Setting the top, right, bottom, and left properties will cause the element to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.


Absolute

position: absolute removes the element completely from the normal flow and positions it relative to the closest ancestor that has a position (relative, absolute, or fixed). If no such ancestor exists, it uses the entire page as its reference. Absolute elements can overlap other elements and do not take up space.


Fixed

position: fixed keeps an element stuck in the same place on the screen, even when you scroll. It is positioned relative to the browser window (viewport). Since it is removed from the normal flow, it can overlap other elements. This is commonly used for sticky navbars, floating buttons, and banners.


Sticky

position: sticky acts like a mix of relative and fixed. Initially, it behaves like relative and moves with the page. Once the element reaches a specific scroll position (like top: 0 or top: 50px), it becomes fixed and sticks to that position. It is mainly used for sticky headers, menus, and scroll sections.
